Twitter Topic Summarization using Speech Acts and Sequential Summarization Title

Click this Link for more info.

In a micro-blog such as twitter, there are a wide array of tweets on each topic of interest according to the user.

In order to enable users to quickly view a summary about a topic searched, a system is proposed in which it involves two major steps.

1. Speech-act based classiﬁcation of the tweets
2. Generating the summary with the classiﬁed tweets

In this approach, given an input query topic, the best relevant tweets are retrieved by the system using tweepy API. Then speech act recognition is done, which is a multiclass classification problem solved using the word based linguistic features and features based on twitter. The Speech Act Recognition recognizes speech acts such as statement, question, comment/suggestion and the rest are tagged as miscellaneous. These four type of speech acts are the most common types found in twitter tweet stream. The classification is done using Logistic Regression after testing that it worked better than SVM. The unwanted/miscellaneous tweets are ignored and then an abstract summary is formed using the remaining tweets. Phrase Extraction and Entity Extraction is done to each tweet and the necessary information is used. Information about entities used in the summaries are stored in a database to avoid redundant information. Summarization is done based on k means algorithm with which an abstract summary is formed which is based on the uniqueness of the tweet and the similarity and various other features which would be used in forming a valid summary. The Summarization is carried out by gathering information from the most informative, time-based and novel tweets from each cluster formed through k means algorithm. Novelty is identified using IDF values and other such measures.

  • Technologies used: Python, Octave
  • Date: December 2014 - April 2014

Pictionary with JIBO

Click this link for more info

Source Code: Click this link

First, ask Jibo to take a guess your drawing. It will take a picture and send it off to be classified by our custom-trained tensor flow neural networks. The first step uses openCV to detect the playing area within the board and crop off the rest of the image, and make it black and white. The image then gets passed off the our tensorflow model to be classified. We build trained neural network using hundreds of images pulled automatically from Bing Image Search. We used the “line drawing” category to limit it to pictures that look more like something drawn a whiteboard. We then supplemented the training data with a few examples of our own hand-drawn pictures. This network gives us a likelyhood for each possible label, which is sent back to Jibo. Then Jibo will go through these items one by one, listening for a “yes” or “no” from the user. There is an adorable celebration and frustration animation that plays when it gets the answer right or wrong. This continues until it exhausts its entire vocabulary, at which point it insults your artwork in protest!
